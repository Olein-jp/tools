# 簡易ビジュアルリグレッションテスト実装メモ

## 概要

既存の `React + Vite` ベースのツールサイトに、Before/After のスクリーンショットを比較するツールを追加するための実装方針を整理した。

## 現在の構成確認

- ルーティングは `src/App.tsx` で管理している
- ツール一覧は `src/pages/HomePage.tsx` の配列で管理している
- ページ単位の SEO は `src/seo/meta.ts` と `src/seo/Seo.tsx` で定義している
- ビルドは `npm run build` で `dist/` を出力する
- 既存の deploy workflow は FTP ベースだったため、今回の要件に合わせて SSH 鍵を使った転送へ見直しが必要

## 実装方針

### 画面

- 新規ページ `src/pages/VisualRegressionPage.tsx` を追加する
- ツール一覧から `/visual-regression` へ遷移できるようにする
- 画面内で以下を提供する
  - Before / After のファイル選択
  - ドラッグ＆ドロップ
  - 左右プレビュー
  - 差分表示 ON/OFF
  - 比較実行
  - リセット
  - エラー表示
  - 比較中インジケーター

### 差分比較

- ブラウザ内で画像を読み込み、Canvas API から `ImageData` を取得する
- `pixelmatch` を使って差分ピクセルを算出する
- 差分結果はオーバーレイ用の PNG Data URL に変換し、After画像の上に重ねる
- 横幅が一致していれば比較可能とする
- 縦幅が異なる場合は、比較用データを長い方の高さにそろえ、不足領域も差分として扱う

### バリデーション

- MIME type で `PNG / JPEG / WebP` のみ許可する
- 1ファイル 10MB 超はエラーにする
- 横幅が一致しない画像はエラーにする
- 読み込み失敗時はユーザー向けメッセージを表示する

## デプロイ方針

- `push` on `main` で workflow を起動する
- `ubuntu-latest` 上で `npm ci` と `npm run build` を実行する
- SSH 鍵を `~/.ssh/id_ed25519` に配置する
- `ssh-keyscan` で `known_hosts` を生成する
- `rsync` を使って `dist/` を `XSERVER_DEPLOY_PATH` へ同期する
- `.htaccess` と Git 関連ファイルは除外する
